<div class="pill" aria-hidden="true">Chapter 1</div>

<h2>Understanding GPT‑5</h2>

<p class="note">Purpose: get familiar with GPT‑5’s capabilities, modes, and strengths.</p>


<article class="lesson" id="l11">

<header>

<h3>Lesson 1.1 — What’s New in GPT‑5</h3>

<span class="meta">10–15 min • overview</span>

</header>

<p>GPT‑5 adds a <strong>reasoning (“Thinking”)</strong> capability that kicks in for multi‑step logic and complex planning. It improves instruction following, reduces hallucinations, and handles longer contexts more reliably. In practice, this means better results for research synthesis, code refactoring, data analysis, and multi‑stage content creation.</p>

<div class="callout"><strong>Key shifts:</strong>

<ul class="list">

<li>Deeper reasoning when tasks are ambiguous or multi‑stage</li>

<li>Cleaner adherence to format and style constraints</li>

<li>More consistent citation and verification prompts</li>

</ul>

</div>

<h4>Quick demo prompts</h4>

<ul class="list">

<li><span class="kbd">Plan → Draft → Refine:</span> “Create a 3‑section blog outline, draft it in 500 words, then refine for skim‑readers with bullets.”</li>

<li><span class="kbd">Evidence mode:</span> “Summarize the key claims <em>and</em> list what a fact‑check would verify.”</li>

</ul>

</article>


<article class="lesson" id="l12">

<header>

<h3>Lesson 1.2 — Choosing the Right Mode</h3>

<span class="meta">10 min • decision rules</span>

</header>

<p>Use <strong>Chat</strong> for speed and drafting; use <strong>Thinking</strong> when the task requires careful step‑by‑step logic, long context juggling, or non‑trivial coding. When in doubt, ask GPT‑5 to decide and justify the choice.</p>

<h4>Heuristics</h4>

<ul class="list">

<li><strong>Chat:</strong> social posts, ad copy, simple Q&A, small edits</li>

<li><strong>Thinking:</strong> multi‑step plans, architecture decisions, data cleaning, refactoring</li>

<li><strong>Switch:</strong> if results feel shallow, ask: “Re‑attempt using deeper reasoning; show steps, then final.”</li>

</ul>

<h4>Meta‑prompt</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">Decide whether to use Chat or Thinking for this task. Explain your choice in 2 lines. Then do the work.</pre>

</article>


<article class="lesson" id="l13">

<header>

<h3>Lesson 1.3 — Prompt Basics That Work</h3>

<span class="meta">15–20 min • patterns</span>

</header>

<p>Great prompts combine <em>constraints</em> + <em>structure</em> + <em>samples</em>. Aim for predictable outputs you can reuse.</p>

<h4>Patterns</h4>

<ul class="list">

<li><strong>CRISP:</strong> <em>Context → Role → Intent → Structure → Parameters</em></li>

<li><strong>Loop:</strong> plan → draft → critique → refine</li>

<li><strong>Guardrails:</strong> “If info is uncertain, flag it and propose how to verify.”</li>

</ul>

<h4>Reusable templates</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">Write a {format} for {audience}. Tone: {tone}. Length: {length}. Include: {checklist}. Finish with a 5‑item action list.</pre>

<pre class="kbd" style="display:block;white-space:pre-wrap">Think step‑by‑step. First outline, then produce the final. If a step is ambiguous, ask for a reasonable assumption and continue.</pre>

</article>


<details>

<summary>Lab A — Chat vs. Thinking (25 min)</summary>

<ol class="list">

<li>Pick a familiar topic.</li>

<li>Run: “Draft a 400‑word explainer for beginners.” in <strong>Chat</strong>.</li>

<li>Re‑run with: “Now solve with deeper reasoning and show steps first.”</li>

<li>Compare depth, structure, and correctness; write a 3‑line rule you’ll use going forward.</li>

</ol>

<p class="note">Deliverable: your personal “When I use Thinking vs Chat” rule card.</p>

</details>

<div class="pill">Chapter 2</div>

<h2>Creating with GPT‑5 (No Code)</h2>

<p class="note">Purpose: use GPT‑5 for content, ideas, and tools without writing code.</p>


<article class="lesson" id="l21">

<header>

<h3>Lesson 2.1 — Everyday Creative Uses</h3>

<span class="meta">20–25 min • content systems</span>

</header>

<p>Stand up repeatable systems for content: ideation → outline → draft → repurpose.</p>

<h4>Workflows</h4>

<ul class="list">

<li><strong>Blog pipeline:</strong> 20 titles → 5 briefs → 1 draft → 5 repurposed posts</li>

<li><strong>Video script kit:</strong> Hook → Promise → Proof → How‑to → CTA</li>

<li><strong>Research digest:</strong> 5 sources → 10 insights → 3 takeaways → action plan</li>

</ul>

<h4>Reusable prompts</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">Create 20 video titles on {topic} for {audience}. Sort by novelty and difficulty. Mark top 5 with ⭐.</pre>

<pre class="kbd" style="display:block;white-space:pre-wrap">Turn this outline into a 90‑second script. Keep sentences tight. Add one surprising fact with a source placeholder.</pre>

</article>


<article class="lesson" id="l22">

<header>

<h3>Lesson 2.2 — Building Assets</h3>

<span class="meta">20 min • landing pages & copy</span>

</header>

<p>Have GPT‑5 generate on‑brand copy and wireframes you can paste into a one‑file website.</p>

<h4>Deliverables</h4>

<ul class="list">

<li>Hero statement + benefits + proof + CTA</li>

<li>FAQ (5–7 items) and objection handling</li>

<li>Policy stubs: privacy, refund, terms (lite)</li>

</ul>

<h4>Prompt</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">Write conversion‑focused landing copy for {offer}. Audience: {who}. Tone: {tone}. Include hero, 3 benefits, social proof, FAQ, CTA. Finish with HTML structure hints (section ids).</pre>

</article>


<article class="lesson" id="l23">

<header>

<h3>Lesson 2.3 — Your Workflow Stack</h3>

<span class="meta">15 min • ops</span>

</header>

<p>Combine GPT‑5 with your everyday tools (Notion, Google Docs, Canva) and enforce consistency with brand rules.</p>

<h4>Brand rules example</h4>

<ul class="list">

<li>Voice: warm, concise, no fluff, xennial humor</li>

<li>Formatting: short paragraphs, bullets, bold benefit labels</li>

<li>Forbidden: generic claims without proof or example</li>

</ul>

<h4>Prompt library starter</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">Rephrase to match brand: warm, concise, confident. Keep under 120 words. End with one action line.</pre>

</article>


<details>

<summary>Lab B — One‑Page Website (45–60 min)</summary>

<ol class="list">

<li>Ask GPT‑5 for copy + <em>single‑file</em> HTML with clean CSS.</li>

<li>Request a publish plan for GitHub Pages/Netlify.</li>

<li>Run a mobile viewport checklist (Lighthouse hints).</li>

</ol>

<p class="note">Deliverable: live landing page URL + checklist.</p>

</details>

<section id="chap3">

<div class="card">

<div class="pill">Chapter 3</div>

<h2>Automating & Extending with GPT‑5</h2>

<p class="note">Purpose: learn basic automation and light development using GPT‑5.</p>


<article class="lesson" id="l31">

<header>

<h3>Lesson 3.1 — GPT‑5 in the API</h3>

<span class="meta">20–30 min • dev basics</span>

</header>

<p>Understand model choices, context limits, and the knobs that trade speed, cost, and depth.</p>

<h4>Cheat sheet</h4>

<ul class="list">

<li><strong>Models:</strong> gpt‑5 (flagship), gpt‑5‑mini (fast/cheap)</li>

<li><strong>Context:</strong> large enough for long briefs & docs; chunk when needed</li>

<li><strong>Controls:</strong> temperature, max tokens, JSON mode, reasoning/verbosity (if available)</li>

</ul>

<h4>Spec prompt</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">You are a systems designer. Produce a JSON spec with: goal, inputs, steps, risks, outputs. Keep keys snake_case. Validate it is parseable JSON.</pre>

</article>


<article class="lesson" id="l32">

<header>

<h3>Lesson 3.2 — Tool Calling & Agents</h3>

<span class="meta">25–35 min • agent pattern</span>

</header>

<p>Design a compact agent: plan → call tools → verify → finalize. Keep tool contracts tight and idempotent.</p>

<h4>Pattern</h4>

<ul class="list">

<li>Planner: produce numbered steps and required tool calls</li>

<li>Executor: call tools, attach results, handle failures</li>

<li>Verifier: cross‑check assumptions, surface uncertainty</li>

</ul>

<h4>Guardrails</h4>

<ul class="list">

<li>Structured outputs (JSON schemas)</li>

<li>Time/size limits; safe fallbacks on error</li>

<li>Logs for each tool call</li>

</ul>

</article>


<article class="lesson" id="l33">

<header>

<h3>Lesson 3.3 — Safe & Reliable Deployment</h3>

<span class="meta">20 min • prod hardening</span>

</header>

<p>Ship small and observable. Add pre‑flight validation, content checks, and a refusal backstop for unsafe requests.</p>

<h4>Deployment checklist</h4>

<ul class="list">

<li>Schema validation + length caps</li>

<li>Retry with backoff; escalate to human review on repeated failure</li>

<li>Unit prompts for critical paths</li>

</ul>

<h4>Release note template</h4>

<pre class="kbd" style="display:block;white-space:pre-wrap">Version {x.y.z}: Added schema checks; reduced temperature to 0.4 for summaries; logged tool_failures with reason field.</pre>

</article>


<details>

<summary>Lab C — Micro‑Agent (60–90 min)</summary>

<ol class="list">

<li><strong>Input:</strong> topic + assets list.</li>

<li><strong>Steps:</strong> outline lessons → draft scripts → create upload checklist → generate course page HTML → summary email.</li>

<li><strong>Output:</strong> a zipped bundle (outline.md, scripts/, page.html, email.txt).</li>

</ol>

<p class="note">Tip: benchmark minimal vs. high reasoning depth and record time/quality trade‑offs.</p>

</details>
